# Device-authenticated billing uses isolated device sessions and device-scoped routes

Billing access is authenticated by a Store Device using device id plus device secret, not by an admin user session. We issue a JWT that identifies the Store Device, load the live device record in middleware as the source of truth for organization/store scope and active status, isolate billing auth from admin auth, and expose billing through device-scoped routes so every billing write is attributed to the authenticated device from draft creation onward. Admin users may still inspect store bills, but they do so through a read-only mode of the shared billing workspace and cannot create or mutate billing data without leaving the admin session and opening a device-authenticated POS session.
